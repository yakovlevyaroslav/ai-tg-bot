# Визитки кода личности

Публичная страница на сайте с кодом личности и разбором **без персональных данных** (имя, дата и место рождения). Пользователь может оплатить визитку в боте или админ может опубликовать её вручную.

## Главная идея

Визитка — **не отдельная таблица** в базе. Это набор полей у пользователя в `users`:

- опубликована ли страница (`visit_card_published`);
- когда опубликована (`visit_card_published_at`);
- очищенный текст для сайта (`visit_card_content`).

**URL нигде не хранится** — он всегда вычисляется из кода личности:

```
{PUBLIC_SITE_URL}/code/{10-значный код}
```

Пример: `https://personality-code.ru/code/3230581078`

Страница открывается **только** если `visit_card_published = true`. Иначе по тому же URL будет 404.

## Где хранится

### Таблица `users`

| Поле | Тип | Назначение |
|------|-----|------------|
| `personality_code` | `TEXT`, UNIQUE | 10-значный код личности; один код — один пользователь |
| `onboarding_data` | `JSONB` | Анкета + разбор (`personality_code_result`, подкоды по системам) |
| `visit_card_published` | `BOOLEAN` | `true` — визитка доступна на сайте |
| `visit_card_published_at` | `TIMESTAMPTZ` | Дата и время публикации |
| `visit_card_content` | `TEXT` | Санитизированный текст разбора для публичной страницы |

### Платежи за визитку

Запись в `pending_payments`:

- `product_type = 'visit_card'`
- `credits_amount = 0` (вопросы не начисляются)
- `rub_amount` — сумма из `VISIT_CARD_PRICE_RUB`

Схема БД: `sql/init.sql`.

## Откуда берётся контент

1. Пользователь проходит анкету в боте.
2. Бот рассчитывает код личности и генерирует разбор (OpenAI).
3. В `onboarding_data` сохраняется:
   - `personality_code` — полный код (10 цифр);
   - `personality_code_result` — текст разбора;
   - `astrology_code`, `human_design_code`, `numerology_code`, `sucai_code`, `jyotish_code` — подкоды для блока на странице.
4. Код дублируется в колонку `personality_code` (с UNIQUE-ограничением).

При публикации вызывается `publishVisitCard(userId)` (`src/shared/db.js`):

1. Берётся `personality_code_result` из `onboarding_data`.
2. Текст прогоняется через `sanitizeVisitCardContent()` (`src/shared/visit-card.js`):
   - имя → «ты»;
   - место рождения → «место рождения»;
   - дата и время → «—».
3. Результат записывается в `visit_card_content`, флаг `visit_card_published = true`.

**Контент визитки — снимок на момент публикации.** Если разбор в анкете изменится, страница обновится только при повторной публикации.

## URL — как формируется и «удаляется»

### Формирование

Функция `buildVisitCardPublicUrl()` в `src/shared/visit-card.js`:

- код должен быть ровно **10 цифр**;
- база URL — `PUBLIC_SITE_URL` из `.env`;
- маршрут сайта: `GET /code/:code` (`src/site/server.js`).

URL **не создаётся в БД** и **не удаляется** как запись. Он всегда один и тот же для данного кода личности.

### «Удаление» URL

При снятии с публикации (`unpublishVisitCard`) обнуляются только флаги и контент визитки. **`personality_code` остаётся.**

Страница по адресу `/code/XXXXXXXXXX` перестаёт открываться (404), но сам путь формально тот же. Если визитку опубликовать снова — URL снова заработает.

## Как создаётся визитка

### 1. Пользователь в боте (платно)

```
Тарифы → Визитка · 1000 ₽ → Получить визитку → ЮKassa
→ webhook или фоновый poll → completeYookassaPayment()
→ publishVisitCard() → ссылка пользователю в Telegram
```

- Цена: `VISIT_CARD_PRICE_RUB` (по умолчанию 1000 ₽).
- До оплаты URL в боте **не показывается**.
- После оплаты — сообщение со ссылкой и кнопка «🪪 Моя визитка» в меню.

Файлы: `src/bot/visit-card.js`, `src/shared/payments.js`, `src/shared/yookassa/service.js`, `src/site/notify.js`.

### 2. Админ в `/admin/visit-cards` (бесплатно)

```
Ввод 10-значного кода → publishVisitCardByCode()
→ поиск пользователя в БД → publishVisitCard()
```

- Платёж **не создаётся**.
- Нужны: пользователь с таким кодом + непустой `personality_code_result`.
- Повторная публикация **обновляет** `visit_card_content`.
- Уведомление пользователю в Telegram **не отправляется**.

### 3. Автоматически после анкеты — нет

Завершение анкеты даёт код и разбор, но **не публикует** визитку. Публикация — только после оплаты или действия админа.

## Как снимается с публикации

`unpublishVisitCard(userId)` в `src/shared/db.js`:

```sql
visit_card_published = FALSE
visit_card_published_at = NULL
visit_card_content = NULL
```

**Где можно снять:**

| Способ | Что ещё происходит |
|--------|-------------------|
| Админка → «Удалить» на `/admin/visit-cards` | Анкета и код пользователя сохраняются |
| `/start` (новая анкета) | Сбрасываются анкета, код, визитка целиком |

После снятия в боте снова показывается «🪪 Визитка · 1000 ₽» в тарифах вместо «🪪 Моя визитка».

## Публичная страница на сайте

- Маршрут: `GET /code/:code`
- Рендер: `src/site/visit-card-page.js`
- Условие показа: `visit_card_published = TRUE` и совпадение `personality_code`
- На странице: код, подкоды по системам, очищенный разбор, ссылка на бота
- Персональных данных нет

Если код не опубликован или указан неверно — страница «Визитка не найдена» (404).

## UX в боте

### До покупки

- Кнопка «🪪 Визитка · {цена} ₽» только внутри **«📋 Тарифы»** (и в `/topup`).
- URL до оплаты **не показывается**.
- Кнопка оплаты: «🪪 Получить визитку».

### После покупки

Кнопка «🪪 Моя визитка» появляется в:

- тарифах (вместо кнопки покупки);
- главном меню после анкеты;
- меню после ответов на вопросы;
- экране баланса (`/balance`).

По нажатию — ссылка и кнопка «🌐 Открыть визитку».

## Админка

**URL:** `/admin/visit-cards` (Basic Auth, как и вся админка)

**Возможности:**

- список всех опубликованных визиток;
- поиск по коду, telegram ID, имени;
- публикация по 10-значному коду;
- снятие с публикации («Удалить»);
- ссылка «Открыть» на публичную страницу.

## Настройки в `.env`

| Переменная | Назначение | По умолчанию |
|------------|------------|--------------|
| `VISIT_CARD_PRICE_RUB` | Цена визитки в боте (₽) | `1000` |
| `PUBLIC_SITE_URL` | База для ссылок вида `/code/...` | — |
| `PUBLIC_BOT_USERNAME` | Ссылка на бота на странице визитки | — |

Пример:

```env
VISIT_CARD_PRICE_RUB=1000
PUBLIC_SITE_URL=https://personality-code.ru
PUBLIC_BOT_USERNAME=YourBot
```

После изменений: `sudo systemctl restart ai-tg-bot ai-tg-site`.

## Ограничения и нюансы

1. **Один код — один пользователь.** UNIQUE на `personality_code`. При коллизии (редко) бот покажет ошибку — написать в поддержку.

2. **URL привязан к коду, не к user_id.** При `/start` код сбрасывается; старая ссылка может перестать работать или когда-нибудь указывать на другого человека с тем же кодом.

3. **Админская публикация бесплатная** — без записи в `pending_payments` и без уведомления в Telegram.

4. **Платёж и визитка связаны логически**, но хранятся в разных полях: флаг у пользователя + опциональная запись в `pending_payments`.

5. **Повторная публикация** (оплата повторно или админ) перезаписывает `visit_card_content` актуальным санитизированным текстом из анкеты.
